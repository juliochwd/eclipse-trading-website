## 2024-05-16 - Non-semantic Elements for Interactive Controls\n**Learning:** Using `<div>` elements for interactive controls like hamburger menus without proper roles and keyboard event handlers prevents screen readers and keyboard users from accessing the functionality.\n**Action:** Replace non-semantic `<div>` with native `<button>` elements for interactive controls, and add `aria-expanded`, `aria-label` attributes and a focus outline to ensure keyboard and screen reader accessibility.
## 2024-05-15 - ARIA Labels for Icon-Only Buttons
**Learning:** Found multiple icon-only interactive elements lacking `aria-label`s across the admin dashboard and landing page (sidebar toggles, notification bell, language switch). Screen readers would announce these as "button" with no context.
**Action:** Consistently added descriptive `aria-label` attributes to icon-only buttons (`#menu-toggle`, `.notification-btn`, `#lang-toggle`) to ensure they are accessible and context-rich for assistive technologies.
